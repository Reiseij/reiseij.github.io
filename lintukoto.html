Lintukoto

Lintukoto on opiskelijoiden ajanhallintaan ja hyvinvointiin keskittynyt sovellus

Hankitut osaamiseni

Asioita, joita opin Lintukoto-sovellusta tehdessäni

Angular

Sovelluksemme Fronend toteutettiin Angularilla. Olin suuressa osassa sovelluksen UI ja UX toteutuksessa. Tein myös sovelluksestamme responsiivisen.

Syncfusionin Scheduler

Sovelluksemme kalenteri toteutettiin Syncfusionin Scheduler komponentilla. Opettelin Schedulerin käytön ja implementoimisen sovellukseemme ja muokkasin siitä juuri meille sopivan

MongoDB Atlas

Backendiin en paljon koskenut, mutta MongoDB:n Atlas tietokantajärjestelmä tuli tutuksi. Kirjoitin tietokantaan dataa ja korjailin virheitä.

Testaaminen

Olin tiimimme päätestaaja. Opin testaamisesta paljon. Testasin jatkuvasti sovelluksen toimivuutta ja raportoin löytmäni bugit. Suurimman osan testauksesta suoritin käsin, mutta tutustuin myös automaatiotestaukseen.

Selenium

Toteutin Seleniumilla automatisoitua käyttöliittymätestausta. Kirjoitin testiskriptejä ja ajoin niitä. Päivitin skriptejäni sitä mukaa kun sovelluksemme eteni.

GIT

Sovelluksemme Versionhallinta toteutettiin Gitillä ja GitHubilla. Opin Gitin perusteet ja GitHubin sekä ZenHubin käytön. Konfliktien sekä pullrequestien tarkistaminen oli yksi töistäni testaajana.

Esisanat

"Lintukoto on opiskelijoille tarkoitettu opintojen hallintatyökalu, joka visualisoi opintojaksot ja tukee opiskelijaa ajanhallinnan kanssa Lintukoto kannustaa opiskelijaa pitämään huolta opintojaksoistaan samalla tavalla kuin hoidetaan virtuaalisia lemmikkejä. Lintukoto myös tukee opiskelijan hyvinvointia ja jaksamista opintojen aikana."
Nämä olivat sovelluksemme lähtökohdat, tälläisen sovelluksen halusimme luoda.

Projektissa minun tehtävänäni oli toimia frontend-koodarina ja päätestaajana. Toteutin myös muutamia visuaalisia elementtejä kuten sovelluksemme logon (oikealla) ja autoin meidän graafikkoa ideoimaan ja mallintamaan.

Projektin alkuvaiheilla suurimmaksi ongelmaksemme huomasimme kalenterin teon. Meitä varoitettiin monesti että kalenterin itse koodaaminen olisi liki mahdottomuus. Tämän vuoksi päädyimme etsimään netin syövereistä jotain kalenteripalikkaa, joka saataisiin suhteellisen vaivattomasti implementoitua sovellukseemme. Löysinkin jo ensimmäisellä viikolla Syncfuisionin Scheduler komponentin, joka vaikutti lupaavalle vaikkakin haastavalle aluksi ymmärtää. Lähdimmme testailemaan komponenttia ja saimmekin jo ensimmäisellä sprintillä karkean kuvan sovelluksemme tulevasta kalenterista:

Lähdin toteuttamaan kalenteriamme ensin tutoriaalien kautta. Tutoriaalien pohjalta ei kuitenkaan saatu täysin meidän näköistä ja meidän tarpeisiimme toimivaa kalenteria. Kalenterin muokkaaminen tapahtui Syncfusionin omien koodinpätkien lisäämisellä ja niiden uudelleen kirjoittamisella.


KALENTERI

Scheduler toimii siten että html-tiedostoon annetaan Syncfusionin ennalta luomia määreitä, jolloin kalenterista saadaan räätälöityä toivotunlainen. Esimerkiksi kun showHeaderBar-määre on asetettu typescriptissä trueksi saadaan kalenteriin näkymään yläpalkki, johon voidaan lisätä kuukausivalikko ja nykyhetkeen siirtymiseen tarkoitettu nappula. Asettamalla trueksi showTimeIndicator-määre saadaan kalenterin päälle viiva havainnollistamaan nykyhetkeä.
Vaikka tämä koodi näyttää yksinkertaiselta niin näiden oikeiden lisäpalikoiden löytäminen ja liittäminen kalenteriin oli aluksi haaste.

SELENIUM

Tein JavaScriptillä muutaman Selenium skriptin. Yksi niistä oli esimerkiksi muistiinpanon testaaminen. Tässä testissä luodaan uusi muistiinpano, varmistetaan että muistiinpano tallentui, muokataan sitä ja lopulta poistetaan se. Tein myös ylimääräisen sleep funktion, jotta näkisin itse prosessin selkeämmin ja jotta videosta tulisi havainnollistavampi. Sleep funktio auttoi myös kun jokin elementti ei latautunut sivulle heti vaan kesti muutaman sekunnin, joten pystyin sleep funktiolla hidastamaan koodin ajoa.

Toinen "ylimääräinen" funktio on päivämäärän muotoilu. En saanut jostain syystä javascriptin omia funktioita toimimaan niinkuin halusin, joten totesin helpommaksi itse formatoida päivämäärän. Halusin päivämäärän siksi, että skripti tunnistaa mitä muistiinpanoa tulee muokata ja poistaa. Jatkokehityksen kannalta ajattelin myös, että jos nämä testit logattaisiin, päivämäärästä voisi ehkä olla hyötyä.

Tutustuin myös hieman XML:ään ja XPathiin. Joitakin elementtejä sivulla oli helpompi löytää XPathin avulla, esimerkiksi Bootstrapin dropdown-valikosta oli vaikea valita tiettyä elementtiä ilman XPathia. 

          
            MUISTIINPANON TESTAUS JAVASCRIPT

            const { Builder, By, Key } = require("selenium-webdriver");

            async function muistiinpanotesti() {
            
              //sleep funktio hidastamaan prosessia
              function sleep(ms) {
                return new Promise(resolve => setTimeout(resolve, ms));
              }

              //webdriver otti väärän muotoisen päivämäärän toLocaleString funktiolla, joten tein päivän muotoilun itse muotoon dd/mm/yyyy
              const d = new Date();
              let dd = d.getDate();
              let mm = d.getMonth() + 1;
              const yyyy = d.getFullYear();
              const min = d.getMinutes();
              const h = d.getHours();
              
              let date = dd + '.' + mm + '.' + yyyy + " " + h + ":" + min;

              const ktunnus = "villev"
              const salasana = "lol"

              const otsikko = "Selenium Testi " + date;
              const muistiinpano = "Tämä on automaattinen Selenium testi. Ajettu: " + date;
              const otsikkoMuokattu = "Selenium Testi muokattu " + date;
              const muistiinpanoMuokattu = "Selenium Testi muokattu " + date;

              // asetetaan firefoxin driveri ja avataan selain
              let driver = await new Builder().forBrowser("firefox").build();

              // siirrytään oikeaan osoitteesen
              await driver.get("http://localhost:4200/");

              //etsitään sivulta tunnuksen ja salasanan kentät ja täytetään niihin oikeat arvot
              await driver.findElement(By.name("tunnus")).sendKeys(ktunnus);
              await driver.findElement(By.name("password")).sendKeys(salasana, Key.ENTER);

              await sleep(2000);

              // etsitään valikosta aktiiviset kurssit nappula ja painetaan sitä
              await driver.findElement(By.partialLinkText("Aktiiviset kurssit")).click();

              // Etsitään aktiiviset kurssit sivulta uuden muistiinpanon lisäys nappi ja painetaan sitä
              await driver.findElement(By.xpath("//button[contains(text(),'Lisää uusi muistiinpano')]")).click();

              // Valitaan otsikko kenttä ja lisätään otsikko
              await driver.findElement(By.name("otsikko")).sendKeys(otsikko);

              // Etsitään ensimäinen kurssi bootstrapin dropdown valikosta ja painetaan sitä
              await driver.findElement(By.name("kurssi")).click();
              await driver.findElement(By.xpath(
                "//li[1]//div[1]//div[1]//div[1]//app-modal[1]//div[1]//div[1]//div[1]//div[2]//form[1]//div[1]//select[1]//option[1]"
                )).click();

              // Etsitään mustiinpanokenttä ja lisätään siihen muistiinpano
              await driver.findElement(By.name("muistiinpano")).sendKeys(muistiinpano);

              // Etsitään muistiinpanon hyväksymisnappi ja painetaan sitä
              await driver.findElement(By.name("muistiinpanoLahete")).click();

              await sleep(2000);

              // Etsitään juuri tehty muistiinpano ja painetaan sitä
              await driver.findElement(By.partialLinkText(otsikko)).click();

              // Etsitään muokkaus nappi ja painetaan sitä
              await driver.findElement(By.className("far fa-edit fa-lg")).click();

              // Etsitään otsikko ja tyhjennetään sen kenttä, ja kirjoitetaan uusi muokattu otsikko
              await driver.findElement(By.xpath("//div[@class='modal-body muokkaus-body']//input[2]")).clear();
              await driver.findElement(By.xpath("//div[@class='modal-body muokkaus-body']//input[2]")).sendKeys(otsikkoMuokattu);

              await sleep(1000);

TYÖNTEKO JA REFLEKTOINTI

Työtunteja on tähän mennessä (24.4) kertynyt 391h. Sovelluksen toteutus ja Ticorporate alkoi 14.1.2020. Ensimmäiset kaksi kuukautta alkoivat lupaavasti ja saimme sovellusta eteenpäin nopeaa tahtia. Kuitenkin vallitsevan tilanteen vuoksi 16.3 lähtien työmme sovelluksen parissa muuttui etätyöskentelyksi. Etätyöt toivat aluksi haasteita, kuten keskittymisen ylläpito sekä tiimin sisäinen kommunikointi. Onneksi olimme alusta lähtien hyödyntäneet Zenhubia työn jakoon ja sprinttien organisointiin. Zenhub auttoi tiimiämme jakamaan työt, joka etenkin etätyöskentelyssä on erityisen tärkeää, kun ei voi vain huikata kavereille, kun aloittaa työstämään jotain.

Töitä tehdessäni, niin etänä kuin paikanpäällä olen oppinut paljon. Toimistolla opin tiimin yhteishengen tärkeydestä ja siitä kuinka tärkeää on kyetä pyytämään apua jos sitä tarvitsee kuitenkaan jatkuvasti häiritsemättä muiden työskentelyä. Etätyöt opettivat itsekurin, työnjaon ja kommunikoinnin tärkeyttä. Asetin itselleni aluksi tavoitteen, että olisin sovelluksen front sekä backend toteutuksessa mukana. Toisaalta onneksi tiimissä oli jo toinen sovelluskehittäjä, joka otti backendin itselleen. Sanoisin osaavani Angularin perusteet nyt niin hyvin, että perusjutuissa ei enää tarvitse googlata kaikkea. Muutenkin frontendin ulkonäkö ja sommittelu hoituu omasta mielestäni ilman suurempia ongelmia. Kehittämiskohteena sanoisin olevan vielä backend. Perusasiat tietokannasta ja backin ja frontin yhteydestä ymmärrän, mutta hankalammat CRUD- ja backend-asiat ovat vielä hakusessa.
